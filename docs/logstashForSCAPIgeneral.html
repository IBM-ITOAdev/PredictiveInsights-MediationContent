Logstash for SCAPI - Background

This document gives some background on using Logstash for mediation with Predictive Insights (PI). For a general understanding of Logstash, see Logstash. I recommend you spend a little time familiarizing yourself with the concepts there. Logstash, whilst having its roots log processing, is a general purpose data transformer. As such it can be used to transform incoming data streams/formats to those needed by PI. A bonus is that the same skills required to use Logstash over with the sister SCALA product, can be applied to PI.

In order to use Logstash for PI mediation you will need these four things

  • Logstash installed (see Logstash website)
  • SCAPI Logstash plug-ins installed
  • the appropriate configuration files - appropriate for your data source and transformation needs
  • a data source e.g. files, perf.mgmt system

Once you have these, working with your data is essentially a matter of modifying the .conf files for your location - typically specifying various directories, and if desired, specifying additional/alternative groups and metrics, following the patterns in the files. Looking at specific mediation examples should be enough to get you going

SCAPI Logstash plug-ins

In addition to base Logstash capability, there are a number of custom plugins which we have created. These provide functionality to make it easier to https://developer.ibm.com/itoa/resources/logstash-plugins/use Logstash with PI, e.g. scacsv which outputs files compliant with PI requirements, as well as plugins specific to format e.g. scabmcfile which encapsulate significant amounts of functionality particular to a a specific data source.

The current set of plugins developed specifically for use with SCAPI are described here

These plugins should be installed on the system where you will be running Logstash

Installation is simply a matter of downloading the plugin code and placing it in an appropriate directory. The latest plug-ins are available here LogstashPlugins repository. Each plugin is prefixed by logstash-, and there are inputs, filters and outputs. Download the entire logstash dir. An easy way from github is to select the 'Download ZIP' button on the right hand side of the page. This will give you a zip file with all the contents. Place the downloaded logstash directory in a location of your choosing e.g.

/path/to/my/logstashPlugins/logstash

At runtime you will reference this plugin path as command line argument when you start Logstash

Configuration files

As is standard with Logstash, configuration files are used to control the behaviour of logstash. These specify the inputs, filters and outputs you wish to use. In the PI context, you will usually use scafile for output, but the inputs and filters will depend on your specific mediation needs.

Specific mediations

Generally,you'll either be using one of the off-the-shelf mediations or building one yourself. For the off-the-shelf ones, the specific documentation for that format should be enough to get your going. If you want to build a mediatiation yourself there are two levels

  • Assembling the desired configuration (.conf) file from existing plug-ins
  • Assembling the desired configuration from existing plug-ins and custom ones you create yourself
The latter is more complicated and involves writing Ruby code. See 'Extending Logstash' for guidance on how to do this. Also examine the existing plugin set, both the core Logstash ones, and the custom SCAPI-related ones for practical examples.
